Необходимо разработать систему синхронизации файлов, аналогичную Dropbox/Google Drive, состоящую из сервера и клиента, написанных на Go.

Сервер

* Хранит корневую директорию синхронизации (server_files).
* Предоставляет REST API по HTTP/HTTPS.
* Выполняет аутентификацию по Bearer Token.
* Возвращает манифест файлов (GET /manifest), содержащий:
    * относительный путь;
    * размер;
    * время последнего изменения;
    * SHA-256 хэш.
* Позволяет скачать файл (GET /download/{path}).
* В дальнейшем будет поддерживать загрузку файлов, удаление, создание директорий и отслеживание изменений.

Клиент

* Хранит локальную директорию (client_files).
* Периодически (polling) запрашивает /manifest.
* Сравнивает локальные файлы с сервером по SHA-256.
* Если файл отсутствует или отличается — скачивает его.
* Создает отсутствующие директории автоматически.
* Пока реализована только односторонняя синхронизация сервер → клиент.

Планируемое развитие

* Двусторонняя синхронизация.
* Загрузка изменений клиента на сервер.
* Удаление файлов и директорий.
* Разрешение конфликтов.
* Передача только изменившихся файлов.
* Возможность использовать fsnotify для мгновенного обнаружения локальных изменений.
* Использование HTTPS и JWT/API Token для авторизации.
* Возможность работы как фоновой службы (Windows Service / launchd на macOS).
* Поддержка нескольких клиентов с независимыми состояниями синхронизации.
* Хранение служебной информации (метаданных, версии файлов, истории) в SQLite или PostgreSQL.
